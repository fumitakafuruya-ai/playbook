# プロンプトの出し方 — AIを使い倒す技術

::: danger 最重要ページ
このページは、このプレイブック全体で最も実践的で、最も差がつく内容です。  
ツールの使い方より、**AIへの指示の出し方**で成果の9割が決まります。
:::

---

## 3つの鉄則

### 1. 抽象的な指示を出すな

**ダメな例：**
```
かっこいいダッシュボードを作って
```

**良い例：**
```
売上データを月別の棒グラフで表示するページを作って。
縦軸は金額（万円）、横軸は2025年1月〜12月。
```

AIは「かっこいい」の定義を知らない。**あなたの頭の中にある完成イメージを、言葉で具体的に伝える**こと。色、サイズ、配置、データの種類——曖昧さをゼロにする。

### 2. 一度に複数の指示を出すな

**ダメな例：**
```
ダッシュボードを作って、グラフも追加して、
デザインも整えて、モバイル対応もして、
あとログイン機能もつけて。
```

**良い例：**
```
まず、売上データを表示するテーブルを作って。
```
↓ 確認して OK なら
```
次に、そのデータを棒グラフにして。
```
↓ 確認して OK なら
```
グラフの色を青系に統一して。
```

**1回1指示。** 出力を確認してから次へ進む。これが最も速い。

### 3. 任せっきりにするな

AIに長い指示を出して放置すると、**あなたの意図とズレた方向に大量のコードが生成される。** 修正コストが膨れ上がり、最終的にゼロからやり直しになる。

正しいやり方：

```
小さく頼む → 確認する → 修正する → 次を頼む
```

**このサイクルを高速で回す。** これがAI開発における PDCA だ。

## 実践：小さく育てる

### ステップ1：骨格を作る

```
「売上管理」というタイトルのページを作って。
中身は空でいい。
```

### ステップ2：パーツを追加する

```
そのページに、月別の売上を入力するテーブルを追加して。
カラムは「月」「売上額」「前年比」の3つ。
```

### ステップ3：見た目を調整する

```
テーブルのヘッダーを紺色背景・白文字にして。
```

### ステップ4：機能を足す

```
売上額を入力したら、前年比が自動計算されるようにして。
```

**毎回、出力を目で確認してから次に進む。** これで迷子にならない。

## お金の使い方

::: warning 課金の鉄則
サブスクリプションの範囲内でやりきる。追加課金はしない。
:::

筆者の実例を紹介する。仲間との会話：

> **仲間：** 月間の制限が上限に達してしまった。どのくらいの利用上限にしておくのがベター？自動にすると青天井でどんどんいっちゃう？
>
> **筆者：** 上限はマックス100ドルか200ドル。追加はすべきではなく、サブスク内でやりきってください。
>
> **仲間：** どんどん追加したくなっちゃって、ちょこちょこ追加してもすぐ行っちゃう……
>
> **筆者：** というか、それがないと睡眠時間がなくなるだけなので、ある意味、健康面で救われているかも。

### なぜ追加課金がダメなのか

追加課金したくなる時点で、**プロンプトの出し方が非効率**な可能性が高い。

| 非効率なやり方 | 効率的なやり方 |
|--------------|--------------|
| 抽象的に頼んで、やり直しを繰り返す | 具体的に頼んで、一発で通す |
| 大きな指示を出して、長時間待つ | 小さな指示を高速で回す |
| AIに任せっきりで放置する | 毎回確認して軌道修正する |
| 制限が来たら追加課金する | 制限が来たら今日は終わり |

**制限は敵ではない。** むしろ、プロンプトの質を高める強制装置であり、深夜の暴走を止めてくれる健康のガードレールだ。

## よくある失敗パターン

### 失敗1：「全部やって」症候群

```
ECサイトを作って。商品一覧、カート、決済、
会員登録、メール通知、管理画面、全部お願い。
```

→ AIは頑張って全部作ろうとするが、**全てが中途半端**になる。まず商品を1つ表示するところから始めよ。

### 失敗2：「いい感じにして」症候群

```
デザインをいい感じにして。
```

→ 「いい感じ」はあなたの頭の中にしかない。**具体的に**：余白を広げる？色を変える？フォントを大きくする？

### 失敗3：「さっきの続き」症候群

```
さっきの続きをやって。
```

→ AIは文脈を覚えているが、**あなたの期待は覚えていない。** 何の続きで、次に何をしてほしいのか、毎回明確に言う。

## プロンプトのテンプレート

迷ったらこの型を使う：

```
【何を】○○のページに
【どこに】テーブルの下に
【何を追加】棒グラフを追加して
【詳細】縦軸は売上（万円）、横軸は月、色は青
```

慣れてきたら一文にまとめる：

```
売上テーブルの下に、月別売上の棒グラフ（青、縦軸万円）を追加して。
```

## まとめ

| 原則 | 実践 |
|------|------|
| 抽象的な指示を出すな | 色・サイズ・配置・データを具体的に |
| 一度に複数頼むな | 1回1指示、確認してから次へ |
| 任せっきりにするな | 高速PDCAで小さく育てる |
| 追加課金するな | サブスク内でやりきる |
| 制限を恐れるな | 制限はプロンプト改善の強制装置 |

> **AIは優秀な部下と同じだ。曖昧な指示を出せば曖昧な成果が返ってくる。具体的な指示を出せば、期待を超える成果が返ってくる。**
