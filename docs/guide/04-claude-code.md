# 第4章：Claude Codeと開発する

## Claude Codeとは

**Claude Code**は、AnthropicのAI「Claude」をVS Codeのターミナルから使えるツールです。

普通のチャットAIとの違い：

| 普通のチャットAI | Claude Code |
|----------------|-------------|
| コードをコピペして渡す | プロジェクト全体を自動で読む |
| 出力をコピペして貼る | ファイルを直接編集してくれる |
| 「このエラーは？」と毎回聞く | エラーを見て自分で修正する |
| 1回の会話で終わり | プロジェクトの文脈を覚えている |

つまり、**隣に座っている優秀なエンジニア**のように働きます。

## セットアップ

### 1. Claude Codeをインストール

VS Codeのターミナルで：
```bash
npm install -g @anthropic-ai/claude-code
```

### 2. 起動

プロジェクトフォルダで：
```bash
claude
```

これだけでClaude Codeが起動し、プロジェクトの全ファイルを読んで文脈を把握します。

## 効果的な指示の出し方

### 良い指示の3原則

**1. 何を作りたいか（What）を明確に**
```
× 「いい感じのダッシュボードを作って」
○ 「今月の売上合計と、前月比を表示するダッシュボードを作って」
```

**2. 既存の情報を渡す**
```
○ 「このNotionページの設問リストをもとに、サーベイアプリを作って」
○ 「data/フォルダにある家賃入金一覧表を分析するアプリを作って」
```

**3. 段階的に依頼する**
```
1回目：「まず記事の一覧を表示するページを作って」
2回目：「記事の新規作成フォームを追加して」
3回目：「投稿カレンダーを追加して」
```

一度に全部頼むより、少しずつ確認しながら進める方が良いものができます。

## CLAUDE.md — AIへの引き継ぎ書

各プロジェクトのルートに `CLAUDE.md` というファイルを置きます。これはClaude Codeが最初に読む「引き継ぎ書」です。

```markdown
# プロジェクト名
> バージョン: v0.1 | 開始: 2026-04-03

## 概要
このプロジェクトは○○を行うアプリです。

## 技術スタック
- Next.js + TypeScript + Tailwind CSS

## 起動方法
npm run dev

## 作業ルール
- ○○は変更しないこと
- デプロイ前にnpm run buildで確認
```

これがあると、新しい会話でも**毎回説明し直す必要がない**のです。

## スキル — よく使う操作を1コマンドに

「プロジェクトの状態を確認して」と毎回打つのは面倒です。

**スキル**を登録すると、`/project-status` の1コマンドで実行できます。

```
/project-status    → 全プロジェクトのGit状態を一覧表示
/ohayo             → 朝のブリーフィング
/daily-plan        → 今日のタスク一覧
```

毎日やる操作をスキルにすると、Claude Codeが**秘書**のように動きます。

## Claude Codeとの付き合い方

### やってもらうべきこと
- コードの作成・修正
- エラーの原因調査と修正
- ファイルの整理・リネーム
- Git操作（コミット、プッシュ、同期）
- ドキュメント生成

### 自分で判断すべきこと
- **何を作るか**（ビジネス判断）
- **何を優先するか**（経営判断）
- **誰に見せるか**（公開範囲）
- **いつリリースするか**（タイミング）

AIは「どう作るか」の実行力。「何を作るか」は経営者の仕事です。
