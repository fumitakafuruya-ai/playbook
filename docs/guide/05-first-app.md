# 第5章：最初のアプリを作る

## 目標

この章では、**1時間で動くWebアプリを1つ作ります。**

作るもの：シンプルなタスク管理アプリ（ToDoリスト）

完成すると：
- ブラウザでタスクを追加・完了・削除できる
- データがローカルに保存される
- VS Codeの中で全て完結する

## ステップ1：プロジェクトを作る（5分）

VS Codeのターミナルで：

```bash
cd D:\fumitaka\Dropbox\dev
npx create-next-app@latest my-first-app
```

いくつか質問されます。全てデフォルト（Enterキー）でOKです。

```bash
cd my-first-app
npm run dev
```

ブラウザで `http://localhost:3000` を開くと、Next.jsの初期ページが表示されます。

**これだけで、Webアプリが動いています。**

## ステップ2：Claude Codeに作ってもらう（30分）

ターミナルで `claude` を起動して、こう伝えます：

```
このNext.jsプロジェクトで、シンプルなタスク管理アプリを作ってください。

機能：
- タスクを追加できる（テキスト入力 + 追加ボタン）
- タスクを完了にできる（チェックボックス）
- タスクを削除できる（×ボタン）
- データはlocalStorageに保存（リロードしても消えない）

デザインはTailwind CSSでシンプルに。日本語で。
```

Claude Codeが自動でファイルを編集し、数分でアプリが完成します。

## ステップ3：GitHubに保存する（5分）

```
コミットして、GitHubにプッシュしてください。
```

Claude Codeが：
1. `git add` でファイルをステージ
2. `git commit` でコミット（メッセージも自動で書く）
3. GitHubにリポジトリを作成
4. `git push` でアップロード

## ステップ4：改良する（20分）

動くものができたら、機能を追加してみましょう：

```
タスクに期限（日付）を追加してください。
期限が過ぎたタスクは赤く表示してください。
```

```
タスクを「仕事」「個人」「買い物」のカテゴリで分類できるようにしてください。
```

**1つずつ依頼 → 確認 → 次の依頼** のリズムで進めるのがコツです。

## この経験から学んだこと

| 学び | 意味 |
|------|------|
| `npx create-next-app` | プロジェクトの雛形を一瞬で作れる |
| `npm run dev` | 開発サーバーを起動する呪文 |
| localhost:3000 | 自分のPCで動いているアプリのアドレス |
| Claude Codeに説明する | コードを書くのではなく、**何が欲しいか伝える** |
| コミット＆プッシュ | 作ったものを安全に保存する |

## 応用：自分のビジネスに使うアプリ

タスク管理アプリは練習です。本番はこうなります：

```
「家賃入金一覧のExcelを読み込んで、
 物件ごとの入金状況を一覧表示するアプリを作って」
```

```
「毎月のフォーラムサーベイの回答を
 Googleスプレッドシートから取得して
 グラフで可視化するダッシュボードを作って」
```

**Excelで手作業していること** → **アプリにする候補** です。
