# 第1章：なぜ自分で作るのか

## 既存ツールの限界

多くの経営者はExcelや業務ソフトでデータ管理をしています。これは正しいスタートです。

しかし、ある段階で壁にぶつかります：

- **やりたいことがツールの機能にない**
- データが複数の場所に散らばる
- 手作業のコピペが増える
- 「これを自動化したい」と思う瞬間が増える

これは「ツールの使い方が悪い」のではなく、**あなたのビジネスがツールの想定を超えた**ということです。

## 「作る」という選択肢

2025年以降、経営者にとって「自分で作る」ハードルは劇的に下がりました。

**Before（2024年以前）:**
```
やりたいこと → エンジニアに依頼 → 見積り → 開発 → 3ヶ月後に完成
```

**After（2025年以降）:**
```
やりたいこと → Claude Codeに説明 → 数時間で動くものが完成
```

コードを「書く」必要はほぼありません。**何を作りたいか説明する力**があれば、AIが書いてくれます。経営者は「何を作るか」を決めるのが仕事。これは今までと同じです。

## 実例：2週間で8プロジェクト

筆者が実際に2週間で立ち上げたプロジェクト：

| プロジェクト | 何をするか | 置き換えたもの |
|------------|-----------|--------------|
| 業務管理アプリ | 日常業務の統合管理 | Excel + Notion + 手帳 |
| Hibana | SNS投稿の計画と分析 | Notion + 手動集計 |
| 自社管理アプリ | 自社の各種管理業務 | Power BI |
| EO Survey | 組織サーベイの収集と可視化 | Googleフォーム + Canva |
| ライフライン | 人生年表のビジュアル化 | PowerPoint |
| 5% Update | 月次振り返りの分析 | 手書きメモ |
| 取引先分析 | 取引先の売上・商品分析 | Power BI |

**共通点：** どれも「既存ツールでやっていたが、限界を感じていたもの」です。

## 必要な投資

### まず始めるなら（通常プラン）

| 項目 | コスト |
|------|--------|
| VS Code | 無料 |
| GitHub | 無料（個人利用） |
| Claude Code | 月額 $20（税込 $22）Pro Mode |
| Supabase（データベース） | 無料（Free Plan） |
| Vercel（公開する場合） | 無料プランあり |
| **合計** | **月額約 $20（約3,000円）** |

### 筆者の実際のコスト

| 項目 | コスト |
|------|--------|
| Claude Code | 月額 $100（税込 $110） |
| Supabase（データベース） | 月額 $25（Pro Plan） |
| **合計** | **月額約 $135（約2万円）** |

### アップグレードの考え方

$20プランで使用上限に達したら$100プランへ。それでも足りなければ$200プランへ。ただし、追加課金の前に[プロンプトの出し方](/guide/prompt-craft)を見直すこと。工夫次第でコストは大幅に減らせます。

### コスト感を白金のランチで例えると

| プラン | 月額 | イメージ |
|-------|------|---------|
| $20 | 約3,000円 | 白金のランチ1回分 |
| $100 | 約15,000円 | ちょっと高めのイタリアンフルコース |
| $200 | 約30,000円 | ミシュラン星ランクの高級フレンチ（アピシウスクラス） |

エンジニアに1つのアプリを依頼するよりも、はるかに安く、はるかに速く、そしてはるかに自由です。
