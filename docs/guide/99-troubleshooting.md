# 付録：トラブルシューティング

## Git関連

### 「コミットできない」

**症状：** コミットボタンがグレーアウト、またはエラーが出る

**原因と対処：**
| 原因 | 対処 |
|------|------|
| メッセージが空 | コミットメッセージを入力する |
| 変更がステージされていない | 「+」ボタンでファイルをステージする |
| 何も変更していない | ファイルを編集してから再度試す |

### 「プッシュできない」

**症状：** 「rejected」エラーが出る

**原因：** 別のPCで作業した内容がGitHubにあり、このPCにはまだ取り込んでいない

**対処：**
1. まず「プル」（同期ボタン）を押す
2. その後にプッシュ

Claude Codeに「同期して」と言えば自動でやってくれます。

### 「VS Codeに変更が残り続ける」

**症状：** コミットしたのに、ソース管理に変更が表示されたまま

**対処：** ソース管理パネルの更新ボタン（🔄）をクリック。それでも消えない場合は `Ctrl + Shift + P` → 「開発者: ウィンドウの再読み込み」

## Claude Code関連

### 「claudeコマンドが見つからない」

**対処：**
```bash
npm install -g @anthropic-ai/claude-code
```
ターミナルを再起動してから `claude` を実行。

### 「Claude Codeがプロジェクトの内容を理解していない」

**原因：** CLAUDE.mdがない、またはSPEC.mdがない

**対処：** プロジェクトのルートにCLAUDE.mdを作成する。「CLAUDE.mdを作って」とClaude Codeに頼めば自動で作ってくれます。

### 「途中で止まった／応答がない」

**対処：**
1. `Ctrl + C` で中断
2. `claude` で再起動
3. 途中までの作業は保存されているので、続きを依頼する

## npm / Node.js 関連

### 「npm: command not found」

**原因：** Node.jsがインストールされていない

**対処：** https://nodejs.org/ からLTS版をダウンロードしてインストール

### 「npm run dev でエラー」

**よくある原因：**
| エラーメッセージ | 対処 |
|----------------|------|
| `MODULE_NOT_FOUND` | `npm install` を実行 |
| `EADDRINUSE` | ポート3000が使用中。別のターミナルで既に起動している |
| `build failed` | Claude Codeに「ビルドエラーを直して」と依頼 |

## 困ったときの最終手段

**Claude Codeに聞く。**

```
エラーが出ました。直してください。
```

スクリーンショットを貼る必要もありません。Claude Codeはターミナルのエラーログを自動で読みます。

それでも解決しない場合：
1. エラーメッセージをコピー
2. Google検索
3. Stack Overflowの回答をClaude Codeに見せて「これを適用して」

## 心構え

> **エラーは敵ではなく、道案内です。**

エラーが出るのは「何かが間違っている」というコンピュータからのメッセージです。恐れる必要はありません。Claude Codeが解読してくれます。

> **壊してもGitがある。**

Gitでコミットしていれば、いつでも前の状態に戻れます。だから安心して実験してください。
